266 /*
267  @description
268  Connection configuration to a remote OPC server
269  @field server_url
270  Zero-terminated path to server URL
271  @field security_policy
272  The chosen OPC-UA security policy for the connection, one of the SOPC_SecurityPolicy_*_URI string
273  @field security_mode
274  The chosen OPC-UA security mode for the connection, one of the OpcUa_MessageSecurityMode constant
275  @field disable_certificate_verification
276  Uses a PKIProvider which does not verify the certificates against a certificate authority.
277  Setting this flag to not 0 severely harms security. The certificate authority is not required in this case.
278  Other certificates are still required when using an encrypted or signed secure channel.
279  @field path_cert_auth
280  Zero-terminated path to the root certificate authority in the DER format
281  @field path_cert_srv
282  Zero-terminated path to the server certificate in the DER format, signed by the root certificate authority
283  @field path_cert_cli
284  Zero-terminated path to the client certificate in the DER format, signed by the root certificate authority
285  @field path_key_cli
286  Zero-terminated path to the client private key which is paired to the public key signed server certificate,
287  in the DER format
288  @field path_crl
289  Zero-terminated path to the certificate revocation list in the DER format
290  @field policyId
291  Zero-terminated policy id. To know which policy id to use, please read a
292  GetEndpointsResponse or a CreateSessionResponse. When username is NULL, the
293  AnonymousIdentityToken is used and the policy id must correspond to an
294  anonymous UserIdentityPolicy. Otherwise, the UserNameIdentityToken is used
295  and the policy id must correspond to an username UserIdentityPolicy.
296  @field username
297  Zero-terminated username, NULL for anonymous access, see policyId
298  @field password
299  Zero-terminated password, ignored when username is NULL. Password is kept in memory for future reconnections.
300  @field publish_period_ms
301  The requested publish period for the created subscription (in milliseconds)
302  @field n_max_keepalive
303  The max keep alive count for the subscription. When there is no notification to send, the subscription waits
304  at most the number of publish cycle before sending a publish response (which is then empty).
305  @field n_max_lifetime
306  The max number of time that a subscription may timeout its publish cycle without being able to send a
307  response (because there is no publish request to answer to). In this case, the subscription is killed by the
308  server. A large value is recommended (e.g. 1000).
309  @field data_change_callback
310  The callback for data change notification
311  @field timeout_ms
312  Connection timeout (milliseconds)
313  @field sc_lifetime
314  Time before secure channel renewal (milliseconds). A parameter larger than 60000 is recommended.
315  @field token_target
316  Number of tokens (PublishRequest) that the client tries to maintain throughout the connection
317  @field generic_response_callback
318  The callback used to transmit generic responses to request passed
319  through SOPC_LibSub_AsyncSendRequestOnSession.
320  */

322 {
323  SOPC_LibSub_CstString server_url;
324  SOPC_LibSub_CstString security_policy;
325  OpcUa_MessageSecurityMode security_mode;
326  uint8_t disable_certificate_verification;
327  SOPC_LibSub_CstString path_cert_auth;
328  SOPC_LibSub_CstString path_crl;
329  SOPC_LibSub_CstString path_cert_srv;
330  SOPC_LibSub_CstString path_cert_cli;

332  SOPC_LibSub_CstString policyId;
333  SOPC_LibSub_CstString username;
334  SOPC_LibSub_CstString password;
335  int64_t publish_period_ms;
336  uint32_t n_max_keepalive;
